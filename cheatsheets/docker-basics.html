Docker Basics

Kontenery · obrazy · wolumeny · sieci · Dockerfile · compose

Docker — jak to dziala?
Image (Obraz)
Szablon tylko-do-odczytu. Zawiera OS, zaleznosci, kod aplikacji. Budowany z Dockerfile.
Container (Kontener)
Uruchomiona instancja obrazu. Izolowany proces z wlasnym FS, siecia, PID. Lekki i efemeryczny.
Volume (Wolumin)
Trwale dane poza kontenerem. Przezywa restart i usuwanie kontenerow.
Network (Siec)
Kontenery komunikuja sie przez wirtualne sieci. Domyslnie bridge, mozna tworzyc wlasne.
Kontenery — uruchamianie
# Uruchom kontener
docker run nginx
# Interaktywny shell w kontenerze
docker run -it ubuntu /bin/bash
# W tle + port mapping + nazwa
docker run -d -p 8080:80 --name moj-nginx nginx
# Ze zmiennymi srodowiskowymi
docker run -e MYSQL_ROOT_PASSWORD=secret mysql
# Z woluminem
docker run -v ./dane:/app/data myapp
-d
detached — w tle
-it
interaktywny terminal
-p 8080:80
port hosta:port kontenera
--name
nadaj nazwe kontenerowi
-e KEY=VAL
zmienna srodowiskowa
-v src:dst
bind mount / wolumin
--rm
usun kontener po zakonczeniu
--restart always
auto-restart po crashu/reboot
Zarzadzanie kontenerami
# Lista uruchomionych
docker ps
# Lista WSZYSTKICH (tez zatrzymanych)
docker ps -a
# Zatrzymaj / uruchom / restart
docker stop moj-nginx
docker start moj-nginx
docker restart moj-nginx
# Usun kontener
docker rm moj-nginx
docker rm -f moj-nginx # force (dzialajacy)
# Logi kontenera
docker logs moj-nginx
docker logs -f moj-nginx # follow (na zywo)
# Wejdz do dzialajacego kontenera
docker exec -it moj-nginx /bin/bash
# Statystyki zuzycia zasobow
docker stats
# Szczegoly kontenera (IP, mounts, env...)
docker inspect moj-nginx
Obrazy (Images)
# Pobierz obraz z Docker Hub
docker pull nginx:alpine
# Lista lokalnych obrazow
docker images
# Zbuduj obraz z Dockerfile
docker build -t myapp:1.0 .
# Tagowanie obrazu
docker tag myapp:1.0 user/myapp:latest
# Wypchnij na Docker Hub
docker push user/myapp:latest
# Usun obraz
docker rmi nginx:alpine
nginx:alpine — nazwa:tag. Brak tagu = :latest. Alpine = minimalna wersja (~5MB).
Dockerfile
Przepis na budowanie obrazu — warstwa po warstwie.
# Przykladowy Dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
FROM
Obraz bazowy — zawsze pierwsza instrukcja
WORKDIR
Katalog roboczy w kontenerze
COPY
Kopiuj pliki z hosta do obrazu
RUN
Wykonaj komende podczas buildu
EXPOSE
Dokumentuj port (nie otwiera go!)
CMD
Domyslna komenda przy starcie kontenera
ENV
Zmienna srodowiskowa
ARG
Argument build-time (nie widoczny w runtime)
ENTRYPOINT
Niezmienny punkt wejscia (CMD jako argumenty)
Woluminy (Volumes)
Dane w kontenerze sa efemeryczne — woluminy je utrwalaja.
# Utworz named volume
docker volume create app-data
# Uzyj named volume
docker run -v app-data:/app/data myapp
# Bind mount (katalog hosta)
docker run -v $(pwd)/src:/app/src myapp
docker volume ls # lista woluminow
docker volume inspect app-data
docker volume rm app-data
docker volume prune # usun nieuzywane
Sieci (Networks)
# Utworz siec
docker network create my-net
# Uruchom kontener w sieci
docker run --network my-net --name api myapp
# Kontenery w tej samej sieci widza sie po nazwie
curl http://api:3000 # z innego kontenera w my-net
bridge
Domyslna — izolowana siec na hoscie
host
Kontener uzywa sieci hosta bezposrednio
none
Brak sieci
docker network ls
docker network inspect my-net
docker network rm my-net
Docker Compose
Wiele kontenerow zdefiniowanych w jednym pliku YAML.
# docker-compose.yml
services:
  web:
    build: .
    ports:
      - "8080:3000"
    depends_on:
      - db
  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: secret
    volumes:
      - db-data:/var/lib/postgresql/data
volumes:
  db-data:
docker compose up -d # start w tle
docker compose down # stop + usun kontenery
docker compose logs -f # logi na zywo
docker compose ps # status serwisow
docker compose build # przebuduj obrazy
docker compose exec web bash # shell w serwisie
Sprzatanie
# Usun WSZYSTKO nieuzywane (kontenery, obrazy, sieci, cache)
docker system prune -a
# Usun zatrzymane kontenery
docker container prune
# Usun nieuzywane obrazy
docker image prune -a
# Ile miejsca zajmuje Docker
docker system df
prune -a usuwa WSZYSTKIE nieuzywane zasoby — uzywaj ostroznie na produkcji!
