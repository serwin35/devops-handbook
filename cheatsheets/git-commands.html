Git Commands

Kontrola wersji · branche · merge · remote · historia · cofanie zmian

Git — przeplyw pracy
Working Directory — git add —→ Staging Area — git commit —→ Local Repo — git push —→ Remote Repo
Konfiguracja
# Ustawienia globalne (dla calego systemu)
git config --global user.name "Imie Nazwisko"
git config --global user.email "email@example.com"
# Domyslna galaz
git config --global init.defaultBranch main
# Sprawdz aktualna konfiguracje
git config --list
Konfiguracja: --global = caly user, --local = tylko to repo (nadpisuje global).
Podstawy
# Nowe repo
git init
# Klonuj istniejace repo
git clone https://github.com/user/repo.git
# Status (co sie zmienilo?)
git status
git status -s # skrocony format
# Dodaj do staging
git add plik.txt # konkretny plik
git add . # wszystko
git add -p # interaktywnie (hunk po hunku)
# Commit
git commit -m "opis zmian"
git commit -am "add+commit" # pomija staging (tracked files)
Branche (galezie)
# Lista galezi
git branch # lokalne
git branch -a # wszystkie (+ remote)
# Utworz nowa galaz
git branch feature/login
# Przelacz sie na galaz
git checkout feature/login
git switch feature/login # nowszy sposob
# Utworz i przelacz (jednoczesnie)
git checkout -b feature/login
git switch -c feature/login
# Usun galaz
git branch -d feature/login # safe (merged)
git branch -D feature/login # force
Merge i Rebase
# Merge galezi do main
git checkout main
git merge feature/login
# Rebase (przenosl commity na czubek main)
git checkout feature/login
git rebase main
Merge vs Rebase
Merge — tworzy merge commit, zachowuje historie
Rebase — przepisuje historie, czystsza linia commitow
Zasada: nie rebase'uj publicznych galezi!
# Konflikty merge — po rozwiazaniu:
git add .
git merge --continue
git merge --abort # cofnij merge
Remote (zdalne repo)
# Dodaj zdalne repo
git remote add origin https://github.com/user/repo.git
# Lista remote'ow
git remote -v
# Wypchnij na remote
git push origin main
git push -u origin main # ustaw upstream (pierwszy raz)
# Pobierz z remote
git fetch # pobierz info (bez merge)
git pull # fetch + merge
git pull --rebase # fetch + rebase
# Wypchnij nowa galaz
git push -u origin feature/login
Historia i diff
# Log commitow
git log
git log --oneline # kompaktowy
git log --graph --oneline --all # drzewko
git log -5 # ostatnie 5
# Roznice
git diff # unstaged zmiany
git diff --staged # staged (przed commit)
git diff main..feature # miedzy branchami
# Kto zmieniil ktora linijke
git blame plik.txt
# Szczegoly commitu
git show abc1234
Cofanie zmian
# Cofnij plik ze staging (przed commit)
git restore --staged plik.txt
# Odrzuc lokalne zmiany w pliku
git restore plik.txt
# Popraw ostatni commit (tresc/pliki)
git commit --amend -m "poprawiony opis"
# Cofnij commit (tworzac nowy — bezpieczne)
git revert abc1234
# Reset (niebezpieczne — zmienia historie!)
git reset --soft HEAD~1 # cofnij commit, zachowaj staging
git reset --mixed HEAD~1 # cofnij commit + staging
git reset --hard HEAD~1 # cofnij WSZYSTKO
reset --hard traci dane bezpowrotnie! Uzywaj revert na publicznych branchach.
Stash — schowek
Tymczasowo odloz zmiany na bok (np. zeby przelaczac branche).
# Schowaj zmiany
git stash
git stash -m "opis zmian"
# Przywroc ze schowka
git stash pop # przywroc + usun ze stash
git stash apply # przywroc, zostaw w stash
git stash list # lista schowkow
git stash drop # usun ostatni
git stash clear # usun wszystkie
.gitignore
Pliki/katalogi ignorowane przez Git.
# Typowy .gitignore
node_modules/
.env
*.log
dist/
.DS_Store
.idea/
__pycache__/
Juz sledzony plik? Usun z trackingu: git rm --cached plik
