Filesystem Hierarchy

Struktura katalogow Linux — FHS (Filesystem Hierarchy Standard)

Kluczowe pliki w /etc
/etc/passwd
Lista uzytkownikow — login, UID, GID, shell
/etc/shadow
Hashe hasel (tylko root)
/etc/group
Definicje grup i ich czlonkowie
/etc/fstab
Montowanie systemow plikow przy starcie
/etc/hostname
Nazwa hosta
/etc/hosts
Lokalne mapowanie nazw DNS
/etc/resolv.conf
Konfiguracja DNS
/etc/sudoers
Konfiguracja sudo (edycja: visudo)
/etc/ssh/sshd_config
Konfiguracja serwera SSH
/etc/apt/sources.list
Repozytoria pakietow (Debian/Ubuntu)
Wirtualne systemy plikow
/proc — procfs
Informacje o procesach i jadrze. Nie istnieje fizycznie na dysku.
cat /proc/cpuinfo # info o CPU
cat /proc/meminfo # info o pamieci
cat /proc/version # wersja jadra
ls /proc/$$ # info o biezacym procesie
/sys — sysfs
Urzadzenia, sterowniki, moduly jadra. Interfejs do konfiguracji hardware.
/dev — devtmpfs
Pliki urzadzen — "wszystko jest plikiem".
/dev/sda # pierwszy dysk
/dev/null # czarna dziura (odrzuca dane)
/dev/zero # nieskonczone zera
/dev/urandom # losowe dane
Przydatne komendy
# Miejsce na dysku per partycja
df -h
# Rozmiar katalogow
du -sh /var/log/*
# Zamontowane systemy plikow
mount | column -t
# Znajdz plik
find /etc -name "*.conf"
# Typ pliku
file /bin/bash
# Gdzie jest zainstalowany program
which nginx
whereis nginx
# Drzewo blokow (dyski, partycje, LVM)
lsblk
LVM — Logical Volume Management
Struktura
Dysk fizyczny /dev/sdb1 → Physical Volume (PV) data_vg → Volume Group (VG) data_lv → Logical Volume (LV) /mnt/data → Mount point
PV = fizyczny dysk → VG = pula z PV → LV = wirtualna partycja
Tworzenie
# 1. Physical Volume
pvcreate /dev/sdb1
# 2. Volume Group
vgcreate data_vg /dev/sdb1
# 3. Logical Volume
lvcreate -L 100G -n data_lv data_vg
# 4. Formatuj i zamontuj
mkfs.ext4 /dev/data_vg/data_lv
mount /dev/data_vg/data_lv /mnt/data
Zarzadzanie
# Rozszerz LV o 50GB
lvextend -L +50G /dev/data_vg/data_lv
resize2fs /dev/data_vg/data_lv
# Snapshot
lvcreate -s -n snap01 -L 10G /dev/data_vg/data_lv
Podglad
pvs # lista PV
vgs # lista VG
lvs # lista LV
Pelna struktura
/ Root — poczatek calego systemu plikow ├── /bin Podstawowe binaria (ls, cp, mv, cat, bash) ├── /boot Jadro (vmlinuz), initramfs, GRUB bootloader ├── /dev Pliki urzadzen (/dev/sda, /dev/null, /dev/tty) ├── /etc Konfiguracja systemowa (passwd, fstab, nginx/) ├── /home Katalogi domowe uzytkownikow (/home/serwin) ├── /lib Biblioteki wspoldzielone (*.so) dla /bin i /sbin ├── /media Automatyczne montowanie (USB, CD-ROM) ├── /mnt Reczne montowanie systemow plikow ├── /opt Opcjonalne oprogramowanie (zewnetrzne paczki) ├── /proc Wirtualny FS — informacje o procesach i jadrze ├── /root Katalog domowy uzytkownika root ├── /run Dane runtime (PID files, sockety) — tmpfs ├── /sbin Binaria systemowe (fdisk, iptables, mount) ├── /srv Dane serwisow (www, ftp) ├── /sys Wirtualny FS — info o sprzecie i jadrze ├── /tmp Pliki tymczasowe (czyszczone przy restarcie) ├── /usr Programy uzytkownika, man pages, biblioteki │ ├── /usr/bin Wiekszosc programow uzytkownika │ ├── /usr/lib Biblioteki dla /usr/bin │ ├── /usr/local Oprogramowanie lokalne (make install) │ └── /usr/share Dane niezalezne od architektury └── /var Dane zmienne (logi, bazy, cache, spools) ├── /var/log Logi systemowe (syslog, auth.log, nginx/) ├── /var/cache Cache pakietow (apt, yum) ├── /var/lib Dane aplikacji (mysql, docker) └── /var/www Domyslny root serwera www
