chmod — tryb symboliczny
Skladnia: chmod [kto][operator][uprawnienie] plik
Operatory
+
dodaj dodaje uprawnienie bez ruszania reszty
-
usun usuwa uprawnienie bez ruszania reszty
=
ustaw zastepuje uprawnienia DOKLADNIE podanymi
# dodaj execute dla wlasciciela
chmod u+x skrypt.sh
# usun write dla grupy i innych
chmod go-w plik.txt
# ustaw dokladnie rw-r--r--
chmod u=rw,g=r,o=r plik.txt
# dodaj execute dla wszystkich (a = u+g+o)
chmod a+x skrypt.sh
# rekurencyjnie na caly katalog
chmod -R 755 /var/www/html
chown — zmiana wlasciciela
Skladnia: chown [opcje] uzytkownik[:grupa] plik/katalog
# zmien wlasciciela na www-data
chown www-data plik.php
# zmien wlasciciela i grupe
chown serwin:developers projekt/
# zmien tylko grupe
chown :docker /var/run/docker.sock
# rekurencyjnie
chown -R serwin:serwin ~/projekty/
-R
rekurencyjnie — cale drzewo katalogow
-v
verbose — wyswietla kazda zmiane
--reference=plik
kopiuje wlasnosc z innego pliku
chgrp — zmiana grupy
Dedykowane narzedzie do zmiany grupy pliku/katalogu
# zmien grupe pliku
chgrp docker /var/run/docker.sock
# rekurencyjnie
chgrp -R www-data /var/www/
chgrp to skrot dla chown :grupa

Specjalne bity — setuid, setgid, sticky
setuid
Wykonuje plik jako wlasciciel. s w pozycji x ownera.
chmod u+s plik # lub chmod 4755 plik
setgid
Na katalogu: nowe pliki dziedzicza grupe katalogu. s w pozycji x grupy.
chmod g+s katalog/ # lub chmod 2755
sticky
Tylko wlasciciel/root moze usunac plik. t w pozycji x other. np. /tmp
chmod +t /shared/ # lub chmod 1777
sudo i sudoers
# Debian/Ubuntu
usermod -aG sudo serwin
# RHEL/CentOS/Fedora
usermod -aG wheel serwin
Edytuj TYLKO przez: visudo
# pelny sudo
serwin ALL=(ALL:ALL) ALL
# sudo bez hasla
serwin ALL=(ALL) NOPASSWD: ALL
# tylko wybrane komendy
serwin ALL=(ALL) /usr/bin/systemctl, /usr/bin/apt
# dla grupy (znak %)
%developers ALL=(ALL) NOPASSWD: /usr/bin/docker
Pliki drop-in (bezpieczniejsze)
visudo -f /etc/sudoers.d/serwin
