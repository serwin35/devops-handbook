Od Unix do Linux — historia
Unix (1969) — Bell Labs
Ken Thompson + Dennis Ritchie
Zasady: modularnosc, prostota, tekstowe strumienie danych, "wszystko jest plikiem"
Linux (1991) — Linus Torvalds
Zainspirowany Minixem i zasadami Unixa
Pierwsza wersja: ~10 239 linii kodu, tylko i386
Dzis (6.x): 33+ mln linii, wiele architektur

Kluczowe komponenty
Jadro (Kernel)
Centralny element OS. Dziala w uprzywilejowanym trybie procesora. Planowanie procesow, zarzadzanie pamiecia, system plikow.
Biblioteki systemowe
Standardowe API: fork(), open(), read(), write(). Interfejs miedzy aplikacjami a jadrem.
Narzedzia systemowe
ps, top, free, df, du — podstawowe narzedzia do monitorowania i zarzadzania.
Kernel space vs User space
Przestrzen jadra (Kernel space)
Pelny dostep do sprzetu. Operacje uprzywilejowane. Zarzadzanie zasobami.
Przestrzen uzytkownika (User space)
Ograniczony dostep. Izolacja procesow. Tu dzialaja Twoje aplikacje.
Wywolania systemowe (syscalls)
Kontrolowany interfejs miedzy user space a kernel:
fork, exec, exit
Zarzadzanie procesami
open, read, write
Operacje na plikach
socket, connect, bind
Komunikacja sieciowa
mmap, brk

LVM — Logical Volume Management
Elastyczna warstwa abstrakcji nad fizycznymi dyskami. Pozwala dynamicznie zmieniac rozmiar, laczyc dyski, tworzyc snapshoty.
Struktura
PV
Physical Volume — fizyczny dysk/partycja
VG
Volume Group — pula z PV
LV
Logical Volume — "wirtualna partycja" z VG
Komendy
# Utworz Physical Volume
pvcreate /dev/sdb1
# Utworz Volume Group
vgcreate data_vg /dev/sdb1
# Utworz Logical Volume (100 GB)
lvcreate -L 100G -n data_lv data_vg
Zalety: dynamiczna zmiana rozmiaru, laczenie dyskow, snapshoty, migracja danych online
Uzytkownicy, grupy, uprawnienia
Model wielouzytkownikowy z podzialem uprawnien. Szczegoly w cheatsheet:
# Utworz uzytkownika
sudo useradd -m -s /bin/bash developer
# Utworz grupe i dodaj usera
sudo groupadd developers
sudo usermod -aG developers developer
Typowe uprawnienia
755
rwxr-xr-x — pliki wykonywalne, katalogi
644
rw-r--r-- — zwykle pliki
600
rw------- — pliki prywatne (klucze SSH)

Monitorowanie systemu
Procesy
top -b -n 1  # jednokrotny snapshot procesow
htop  # interaktywny monitor (kolorowy, mysz, sortowanie)
ps aux | grep nginx
htop — lepszy top. Instalacja: sudo apt install htop. Kolorowe drzewo procesow, filtrowanie, kill bezposrednio z UI.
Pamiec
free -h  # human-readable
vmstat 1  # co 1 sekunde
Dysk i siec
df -h  # miejsce na dysku
du -sh /*  # rozmiar katalogow
netstat -tuln  # otwarte porty
Instalacja i podstawowa konfiguracja
Aktualizacja systemu
sudo apt update  # listy pakietow
sudo apt upgrade  # aktualizacja
Niezbedne narzedzia
sudo apt install build-essential git curl wget
Siec
sudo netplan apply
Firewall (UFW)
sudo ufw enable
sudo ufw allow ssh
Metody instalacji: WSL (Windows), VirtualBox / VMware, dual boot, cloud VM (AWS, Azure, DigitalOcean)
