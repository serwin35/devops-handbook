01 — Wprowadzenie do DevOps

DevOps · konteneryzacja · Kubernetes · Linux · Docker · Git · CI/CD

Czym jest DevOps?
Development + Operations
DevOps laczy zespoly programistyczne (Dev) i operacyjne (Ops) — likwiduje bariery miedzy pisaniem kodu a jego wdrazaniem na produkcje.
Infrastructure as Code (IaC)
Cala infrastruktura zarzadzana jest kodem — mozna ja wersjonowac, testowac i automatycznie odtwarzac. Narzedzia: Terraform, Ansible, Pulumi.
CI/CD Pipeline
Continuous Integration / Continuous Delivery — automatyczne budowanie, testowanie i wdrazanie kodu. Zmniejsza failure rate po deployach.
Mikrouslugi
Zamiast jednej wielkiej aplikacji (monolit) — wiele malych, niezaleznych serwisow komunikujacych sie przez API.
Wirtualizacja vs Konteneryzacja
Maszyna wirtualna (VM)
Pelna kopia systemu operacyjnego na hypervisorze
Ciezka (GB), wolny start, pelna izolacja
Narzedzia: VMware, VirtualBox, Hyper-V
Kontener (Docker)
Wspoldzieli jadro hosta, izoluje procesy
Lekki (MB), start w sekundach, mniej overhead
Narzedzia: Docker, Podman, containerd
VM = izolacja na poziomie hardware
Kontener = izolacja na poziomie procesu
Kubernetes (K8s) — orkiestracja
Zarzadza klastrem kontenerow — automatyczny deployment, skalowanie, restart, load balancing.
Pod
Najmniejsza jednostka — jeden lub wiecej kontenerow
Service
Stabilny endpoint sieciowy do grupy Podow
Deployment
Deklaratywny opis stanu — ile replik, jaki obraz
Namespace
Logiczna separacja zasobow w klastrze
Certyfikacja: CKA (Certified Kubernetes Administrator)
Docker — podstawy
Enkapsulacja aplikacji w kontenerze — izolacja procesow od systemu hosta.
# Testowe uruchomienie kontenera
docker run hello-world
# Uruchomienie z interaktywnym shellem
docker run -it ubuntu /bin/bash
# Lista uruchomionych kontenerow
docker ps
# Lista WSZYSTKICH kontenerow (takze zatrzymanych)
docker ps -a
# Zatrzymanie kontenera
docker stop <container_id>
Git — kontrola wersji
Konfiguracja globalna
git config --global user.name "Twoja Nazwa"
git config --global user.email "email@example.com"
Podstawowe komendy
# Inicjalizacja repozytorium
git init
# Stan repo (skrocony format)
git status -s
# Dodaj + commit
git add . && git commit -m "opis zmian"
# Wypchnij na remote
git push origin main
APT — menedzer pakietow
Menedzer pakietow dla dystrybucji Debian/Ubuntu. Wymaga uprawnien root (sudo).
# Pobierz najnowsze listy pakietow
sudo apt update
# Zainstaluj aktualizacje pakietow
sudo apt upgrade
# Instalacja pakietu
sudo apt install nazwa_pakietu
# Usun pakiet (zachowaj konfiguracje)
sudo apt remove nazwa_pakietu
# Usun pakiet + konfiguracje
sudo apt purge nazwa_pakietu
# Wyszukaj pakiet
apt search slowo_kluczowe
--help
Pomoc — skladnia i opcje kazdej komendy
GitHub Actions — CI/CD
Automatyzacja procesow build/test/deploy bezposrednio w repo na GitHub.
# .github/workflows/build.yml
name: Build
on: push
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: echo "Hello CI!"
Workflow uruchamia sie na push, pull_request, schedule lub manual dispatch.
Monitoring i logi
Grafana
Dashboardy do wizualizacji metryk — CPU, RAM, ruch sieciowy, custom metrics
Prometheus
Zbieranie i przechowywanie metryk (time series database)
ELK Stack
Elasticsearch + Logstash + Kibana — centralne logowanie i analiza
Monitoring srodowisk produkcyjnych to kluczowy element pracy DevOps — pozwala reagowac na problemy zanim zauwazyja je uzytkownicy.
